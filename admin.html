查看資料庫建立教學 ／ 查看 GitHub Actions 部署教學 
-- 在 Supabase SQL Editor 執行一次即可
create table wedding_seating_data (
  event_code text primary key,
  data jsonb not null default '{}'::jsonb,
  updated_at timestamptz not null default now()
);
alter table wedding_seating_data enable row level security;

-- 任何人（含未登入的賓客查詢頁）可以讀取
create policy "public can read"
  on wedding_seating_data for select
  using (true);

-- 只有登入帳號（新人自己）可以新增/修改/刪除
create policy "authenticated can insert"
  on wedding_seating_data for insert
  with check (auth.role() = 'authenticated');
create policy "authenticated can update"
  on wedding_seating_data for update
  using (auth.role() = 'authenticated')
  with check (auth.role() = 'authenticated');
create policy "authenticated can delete"
  on wedding_seating_data for delete
  using (auth.role() = 'authenticated');

Project URL 與 anon key 可在 Supabase 專案的 Settings → API 找到。「活動代碼」是你自訂的代號（例如 wedding2026），所有裝置輸入相同代碼即可共用同一份資料。
建立後台登入帳號：到 Supabase 專案的 Authentication → Users，點 Add user，輸入 Email／密碼並直接標記為已驗證（Auto Confirm User），這組帳密就是這個後台頁面的登入帳號。建議到 Authentication → Settings 把「Allow new users to sign up」關閉，避免其他人自行註冊帳號。
這個 repo 裡的 .github/workflows/deploy-pages.yml 會在每次 push 到 main 時自動執行，用 GitHub repo 的 Secrets 產生 config.js 並部署到 GitHub Pages，網頁開啟後會自動讀取並連線，不需要手動輸入上面的欄位。設定步驟：
1. repo 的 Settings → Secrets and variables → Actions，新增三個 Repository secrets：SUPABASE_URL、SUPABASE_ANON_KEY、EVENT_CODE
2. repo 的 Settings → Pages，Source 選「GitHub Actions」
3. push 到 main 分支，到 repo 的 Actions 分頁即可看到執行紀錄；成功後 Pages 網址就會是你的線上版本（index.html 是賓客查詢首頁，admin.html 是需要登入的後台）
本機測試可複製 config.example.js 為 config.js 並填入真實值（已加入 .gitignore，不會被提交）。
